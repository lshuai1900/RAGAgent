# FRONTEND_UI_SPEC.md

> Ragent-Py 前端 UI 设计规范（P1.0）
>
> 本文件定义前端产品的视觉风格、页面清单、路由清单、布局规范与中文文案映射。
> 前端工程实现见 `docs/FRONTEND_ARCHITECTURE.md`，任务拆分见 `docs/FRONTEND_TASK_PLAN.md`。

---

## 1. 前端产品目标

为 Ragent-Py 后端 MVP（知识库管理 + 文档摄取 + 流式问答）提供配套的工程控制台风格 Web UI，覆盖：

1. 知识库的创建、列表、详情查看
2. 文档上传与摄取进度轮询
3. 基于知识库的流式问答（SSE）
4. 后端 API 地址配置

**MVP 边界**：本前端只对接现有后端 API，不引入新的后端能力，不实现多知识库路由、Rerank 配置、用户认证、对话标题生成等暂缓项。

## 2. 参考来源与复刻边界

### 2.1 参考对象
参考 [xerrors/Yuxi](https://github.com/xerrors/Yuxi) 的 RAG 知识库前端**信息架构与交互模式**，重点参考：

- 知识库列表页：卡片网格 + 新建弹窗 + 点击进入详情
- 知识库详情页：顶部基本信息 + 标签页组织功能区
- 文件管理：上传 + 状态统计 + 表格 + 状态轮询
- 聊天问答：选择知识库 + 输入问题 + 流式输出 + 引用来源
- 设置：后端 API 地址配置

### 2.2 复刻原则（硬约束）
- **只参考产品结构与交互模式，不复制 Yuxi 源码**（不复制其组件代码、样式、变量命名、目录结构）。
- 不照搬 Yuxi 的技术栈（Yuxi 使用 React，本项目使用 Vue 3）。
- 视觉风格按本文件第 7 节"工程控制台风格"执行，不模仿 Yuxi 的具体配色与插画。

### 2.3 禁止行为
- ❌ 禁止复制 Yuxi 仓库的任何源代码片段
- ❌ 禁止复用 Yuxi 的组件命名、CSS 类名、设计 Token
- ❌ 禁止引入营销页/落地页风格

## 3. 中文 UI 要求（硬约束）

前端所有**用户可见文案**必须使用简体中文。

### 3.1 必须中文的范围
- 顶部导航 / 侧边菜单
- 页面标题与面包屑
- 按钮（如"新建知识库""上传文档""发送""保存设置"）
- 表单 label、placeholder、校验提示
- loading 文案（如"正在上传…""正在思考…""正在检索…"）
- 成功提示（如"知识库创建成功""文档上传成功"）
- 错误提示（如"接口请求失败""连接后端失败，请检查 API 地址"）
- 空状态（如"暂无知识库，点击新建""暂无文档""暂无对话记录"）
- 状态标签（见第 8 节状态文案映射）
- 表格列名（如"文档名称""文件类型""大小""状态""创建时间""操作"）
- 引用来源说明（如"引用来源""相关度""来自文档"）
- 弹窗标题与确认/取消按钮

### 3.2 保持英文的范围
- 代码变量名、文件名、类型名、接口路径（如 `kb_id`、`/api/v1/chat/sse`）
- 状态机枚举值（如 `pending`、`completed`，仅作为内部数据，展示时映射为中文）
- trace_id（展示时前缀"追踪编号："，值保持原样）

### 3.3 禁止行为
- ❌ 禁止中英混杂的可见文案（如"创建 KB""Upload 文档""Loading…"）
- ❌ 禁止把后端枚举值原样展示给用户（必须经过中文映射）
- ❌ 禁止把英文错误信息原样抛给用户（必须包装为中文提示 + 追踪编号）

## 4. 页面清单

| 编号 | 页面 | 路由 | 说明 |
|---|---|---|---|
| P-01 | 知识库列表页 | `/` | 卡片网格展示所有知识库，支持新建 |
| P-02 | 知识库详情页 | `/knowledge-bases/:kbId` | 顶部基本信息 + 标签页（文件管理/检索测试/聊天问答/配置） |
| P-03 | 文件管理（详情页标签页） | `/knowledge-bases/:kbId?tab=documents` | 上传 + 状态统计 + 文档表格 + 状态轮询 |
| P-04 | 检索测试（详情页标签页） | `/knowledge-bases/:kbId?tab=retrieve` | 输入 query → 返回 top_k 检索结果（MVP 可选实现） |
| P-05 | 聊天问答（详情页标签页） | `/knowledge-bases/:kbId?tab=chat` | 选知识库 + 输入问题 + SSE 流式输出 |
| P-06 | 设置页 | `/settings` | 配置后端 API 地址，保存到 localStorage |

> 说明：检索测试（P-04）依赖后端是否提供独立检索接口。当前后端 MVP **未单独暴露** `/retrieve` 接口（检索在 `chat/sse` 内部完成），因此 P-04 在 P1.5 联调阶段再决定是否启用；若后端无独立检索接口，则该标签页显示"功能开发中"占位，不阻塞 MVP。

## 5. 路由清单

| 路由路径 | 名称 | 组件（建议） | 是否需要参数 |
|---|---|---|---|
| `/` | `knowledgeBaseList` | `views/knowledge-base/List.vue` | 否 |
| `/knowledge-bases/:kbId` | `knowledgeBaseDetail` | `views/knowledge-base/Detail.vue` | `kbId` |
| `/settings` | `settings` | `views/Settings.vue` | 否 |
| `/:pathMatch(.*)*` | `notFound` | `views/NotFound.vue` | 否 |

### 5.1 路由约束
- 使用 Vue Router 4，`createRouter` + `createWebHistory`
- 详情页通过 `route.params.kbId` 获取知识库 ID
- 标签页切换通过 `route.query.tab` 同步（`documents` / `retrieve` / `chat` / `config`），支持刷新保持标签
- 未匹配路由统一跳转 404 页，404 页提供"返回首页"按钮（中文）

## 6. 页面布局

### 6.1 全局布局（Layout）
```
┌──────────────────────────────────────────────────────────┐
│  顶部导航栏（Logo + 标题"RAGAgent 控制台" + 右侧"设置"入口） │
├──────────────────────────────────────────────────────────┤
│                                                          │
│                    <router-view />                       │
│                   （页面主内容区）                          │
│                                                          │
└──────────────────────────────────────────────────────────┘
```
- 顶部导航栏固定（sticky），高度 56px，浅色背景 + 细底边
- 主内容区最大宽度 1200px 居中，左右留白
- 不使用左侧侧边栏（MVP 页面少，顶部导航足够）

### 6.2 知识库列表页（P-01）
```
┌──────────────────────────────────────────────────────────┐
│  知识库                          [ + 新建知识库 ]          │
├──────────────────────────────────────────────────────────┤
│  ┌─────────┐  ┌─────────┐  ┌─────────┐                  │
│  │ KB 卡片  │  │ KB 卡片  │  │ KB 卡片  │                  │
│  │ 名称     │  │ 名称     │  │ 名称     │                  │
│  │ 描述     │  │ 描述     │  │ 描述     │                  │
│  │ 文档数   │  │ 文档数   │  │ 文档数   │                  │
│  │ 状态标签 │  │ 状态标签 │  │ 状态标签 │                  │
│  └─────────┘  └─────────┘  └─────────┘                  │
│                                                          │
│  空状态：暂无知识库，点击"新建知识库"创建                   │
└──────────────────────────────────────────────────────────┘
```
- 卡片网格：`grid` 布局，每行 3 列（响应式：宽屏 3 列、中屏 2 列、窄屏 1 列）
- 每张卡片展示：名称、描述（截断）、文档数、状态标签、创建时间
- 点击卡片 → 跳转详情页
- 右上角"新建知识库"按钮 → 打开新建弹窗（Modal）

### 6.3 新建知识库弹窗
- 表单字段：名称（必填）、描述（选填）、分块策略（默认 recursive）、分块大小（默认 512）、分块重叠（默认 64）
- Embedding 模型与维度：MVP 阶段使用默认值（`text-embedding-v3` / `1024`），表单中**只读展示**或隐藏，避免用户填错导致与后端不一致
- 校验：名称必填且 1-128 字符；分块重叠必须小于分块大小
- 按钮：取消 / 确定（中文）

### 6.4 知识库详情页（P-02）
```
┌──────────────────────────────────────────────────────────┐
│  ← 返回   知识库名称                                      │
│  描述 / 文档数 / 状态 / 创建时间                          │
├──────────────────────────────────────────────────────────┤
│  [ 文件管理 ] [ 检索测试 ] [ 聊天问答 ] [ 配置 ]          │
├──────────────────────────────────────────────────────────┤
│                                                          │
│              （当前标签页内容）                            │
│                                                          │
└──────────────────────────────────────────────────────────┘
```
- 顶部"← 返回"按钮回到列表页
- 标签页使用 Ant Design Vue `Tabs`，默认激活"文件管理"
- 标签页切换同步到 URL query（`?tab=xxx`）

### 6.5 文件管理（P-03）
```
┌──────────────────────────────────────────────────────────┐
│  状态统计：待处理 2 | 处理中 1 | 已完成 8 | 失败 1        │
│                                  [ + 上传文档 ]          │
├──────────────────────────────────────────────────────────┤
│  文档名称      类型   大小     状态      创建时间   操作   │
│  report.pdf    PDF   1.2MB   已完成    2026-06-28  删除   │
│  notes.md      MD    12KB    处理中    2026-06-28  -      │
│  ...                                                     │
└──────────────────────────────────────────────────────────┘
```
- 顶部状态统计：按文档状态分组计数（中文状态文案）
- "上传文档"按钮 → 打开上传弹窗（选择文件 + 确认）
- 文档表格：列名全中文；状态列用 `Tag` 着色（见第 8 节）
- 状态轮询：存在 `pending/parsing/chunking/embedding/indexing` 状态文档时，每 3 秒轮询 `GET /api/v1/documents?kb_id=xxx`；全部进入 `completed/failed` 终态后停止轮询
- 操作列：MVP 仅保留"删除"占位（后端 MVP 无删除接口，按钮置灰或隐藏，避免误调用）

### 6.6 聊天问答（P-05）
```
┌──────────────────────────────────────────────────────────┐
│  当前知识库：[知识库名称 ▼]   top_k: [5]                  │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  用户：什么是 RAG？                                       │
│                                                          │
│  助手：RAG 是检索增强生成……（流式逐字输出）                │
│  引用来源：                                               │
│   • 来自文档 report.pdf  相关度 0.87                     │
│                                                          │
├──────────────────────────────────────────────────────────┤
│  [ 输入问题 ……                           ] [ 发送 ]      │
└──────────────────────────────────────────────────────────┘
```
- 顶部下拉固定为当前知识库（从详情页带入），可切换
- 会话 ID（`session_id`）由前端生成（UUID），每次"新建对话"重置
- 输入框 + 发送按钮；发送后禁用输入，直到 `done` / `error` 事件到达
- 流式输出：收到 `delta` 事件逐字追加到助手消息
- loading 文案：发送后、首个 `delta` 到达前显示"正在思考…"
- 错误处理：收到 `error` 事件 → 中文错误提示 + "追踪编号：{trace_id}"
- session_id 在前端生成（crypto.randomUUID 或 uuid 库）

```
┌──────────────────────────────────────────────────────────┐
│  设置                                                    │

│  后端 API 地址                                            │
```

- "保存"按钮 → 写入 localStorage（key：`ragent.apiBaseUrl`）

## 7. 视觉风格：工程控制台风格

### 7.1 风格定位
- **工程控制台风格**：高信息密度、克制配色、细边框、无装饰
- 面向开发者/管理员，不面向 C 端用户
- 强调可读性、状态可识别性、操作确定性

### 7.2 视觉规则
| 维度 | 规则 |
|---|---|
| 主色调 | 工程蓝（Ant Design Vue 默认主色 `#1677ff`），不自定义品牌色 |
| 背景 | 浅色背景（`#f5f5f5` 内容区 / `#ffffff` 卡片背景） |
| 边框 | 细边框（1px solid `#f0f0f0`），无阴影或极轻阴影 |
| 圆角 | 统一 6px（卡片、按钮、输入框） |
| 字体 | 系统默认无衬线字体栈，正文 14px，标题 16-20px |
| 间距 | 8px 栅格（8/16/24/32） |
| 表格 | 行高紧凑（small size），斑马纹关闭，hover 高亮 |
| 卡片 | 白底 + 细边框，hover 时边框变蓝，无抬升动画 |
| 状态色 | 见第 8 节统一映射，不自定义 |

### 7.3 禁止行为
- ❌ 禁止营销页风格（大图 Banner、渐变背景、插画、口号）
- ❌ 禁止过重阴影、大圆角（>8px）、玻璃拟态
- ❌ 禁止引入额外 UI 框架（Element Plus / Naive UI / Vuetify 等），仅用 Ant Design Vue
- ❌ 禁止自定义一套设计 Token 覆盖 Ant Design Vue 主题（MVP 用默认主题 + 轻量 CSS 调整）

## 8. 状态文案映射

前端**禁止**把后端枚举值原样展示，必须经过以下映射。

### 8.1 文档状态（DocumentStatus）
| 后端值 | 中文文案 | Tag 颜色 |
|---|---|---|
| `pending` | 待处理 | `default`（灰） |
| `parsing` | 解析中 | `processing`（蓝） |
| `chunking` | 分块中 | `processing`（蓝） |
| `embedding` | 向量化中 | `processing`（蓝） |
| `indexing` | 入库中 | `processing`（蓝） |
| `completed` | 已完成 | `success`（绿） |
| `failed` | 失败 | `error`（红） |

> 未知状态兜底：显示"未知状态"，Tag 用 `default`。

### 8.2 知识库状态（KnowledgeBaseStatus）
| 后端值 | 中文文案 | Tag 颜色 |
|---|---|---|
| `active` | 启用 | `success`（绿） |
| `archived` | 已归档 | `default`（灰） |

### 8.3 文件类型展示
| 后端 file_type | 中文展示 |
|---|---|
| `txt` | TXT |
| `md` | Markdown |
| `pdf` | PDF |
| 其他 | 原值大写 |

### 8.4 文件大小展示
- 使用 dayjs / 原生格式化：小于 1KB 显示 `xxx B`，小于 1MB 显示 `xxx KB`，否则 `xxx MB`，保留 1 位小数。

### 8.5 时间展示
- 使用 dayjs，统一格式 `YYYY-MM-DD HH:mm`，不展示秒。

## 9. 交互细节约定

### 9.1 加载态
- 列表/表格加载：使用 `Spin`，文案"加载中…"
- 按钮提交：按钮进入 loading，文案不变或加"…"
- 流式问答首包：助手消息区显示"正在思考…"，首个 `delta` 到达后替换为实际内容

### 9.2 反馈提示
- 成功：`message.success`，中文文案（如"知识库创建成功"）
- 失败：`message.error`，中文文案 + 追踪编号（如"接口请求失败，追踪编号：xxx"）
- 网络错误：`message.error`（"连接后端失败，请检查 API 地址设置"）

### 9.3 确认操作
- 危险操作（如删除）使用 `Modal.confirm`，标题与按钮全中文
- MVP 无删除接口，删除按钮置灰，不触发弹窗

### 9.4 空状态
- 使用 `Empty` 组件，配中文描述文案
- 知识库列表空：`description="暂无知识库，点击右上角新建"`
- 文档列表空：`description="暂无文档，点击右上角上传"`
- 对话记录空：`description="暂无对话记录，输入问题开始提问"`

## 10. 引用来源说明（待后端对齐）

> **已知限制**：当前后端 SSE 事件流（`start` / `delta` / `done` / `error`）**仅携带回答内容**，未在事件中携带检索引用来源（chunk_id / document_id / score）。

### 10.1 MVP 处理策略
- 前端 MVP **优先保证流式回答的正确展示**。
- 引用来源区域：MVP 阶段先渲染 UI 占位（"引用来源"标题 + 空状态），当 `done` 事件未携带引用时，显示"本次回答无引用来源"。
- 若 P1.5 联调阶段后端在 `done` 事件扩展 `citations` 字段，前端再补充渲染逻辑。

### 10.2 引用来源渲染规范（后端扩展后启用）
- 标题："引用来源"
- 每条引用：`• 来自文档 {document_name}　相关度 {score}`
- 相关度展示为 2 位小数（如 `0.87`）
- 点击引用可滚动高亮对应文档（MVP 可选，不强制）

## 11. 与后端的对接边界

- 前端只调用 `docs/FRONTEND_API_CONTRACT.md` 中列出的接口
- 前端不直接访问 PostgreSQL / Milvus / Embedding API / LLM API
- API Base URL 从 localStorage 读取，默认 `http://localhost:8000`
- CORS：当前后端 `main.py` 未配置 CORS 中间件，P1.5 联调阶段需后端补 `CORSMiddleware`（本轮禁止改动后端）

## 12. 不在本规范范围内（暂缓）

以下不在前端 MVP 范围，禁止提前实现：
- 用户登录 / 权限管理
- 多知识库路由选择
- Rerank 模型配置
- 对话标题自动生成
- 文档删除 / 重新摄取
- 暗色主题
- 国际化（i18n，MVP 锁定简体中文）
- 移动端适配（MVP 仅保证桌面端可用）
